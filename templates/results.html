Image Scaling Results

Challenges

  • The initial results of the API evaluation yielded no definitive findings: all the returned scores were 0.
  • To address the API issue, a stronger image scaling attack was used at the expense of the attack's perceptibility.
  • Due to time and funding constraints, the stronger image scaling attack + adversarial attack was not evaluated online.

Image Scaling Attack + Adversarial Attack

  • The weak image scaling attack + adversarial attack do not need to be evaluated with a perceptual similarity metric, as the differences are vast.
  • The robust image scaling attack + adversarial attack looks less perceptible than the plain adversarial attack, but more perceptible than the previous attack combination.
  • The runtime of the robust linear scaling attack averaged 235.36 seconds, and the runtime of the weak linear scaling attack averaged 161.09 seconds (batches of 5 images).

Online APIs

  • In order to extract meaningful results from the API evaluation, a new dataset was constructed using a stronger image scaling attack capable of altering more pixels.
  • While a specific input size was not found, a range was identified.
  • Both APIs utilized linear scaling and MTCNN, though the OpenCV attacks were also effective on AWS.
