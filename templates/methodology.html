Face-Off Methodology

Previous Work

  • Our previous work on Face-Off and adversarial face obfuscation already accomplishes the following:
    - Identifies the challenges with face recognition on social media.
    - Investigates the transferability of adversarial examples generated on face recognition models.
    - Explores the viability of defenses such as adversarial training.
    - Performs longitudinal evaluations against commonly used face recognition APIs (AWS, Azure, Face++).
    - Creates a framework for users to upload images of faces to be masked.
    - Performs user studies on user perceptions of privacy and the quality of the masked face images.
  • While this work has been accepted into the PETS'21 conference, the contributions made for this course project will not be included.
  • Most likely, the insights from this project will be formed into a workshop paper or added to a peer's ongoing research project investigating image scaling and adversarial examples.

Image Scaling Attack + Adversarial Attack

  • By applying an image scaling attack to the noise mask generated by an adversarial attack, the resulting image contains much less visible perturbations.
  • The perturbation artifacts are greatly reduced, since image scaling attacks only require a few pixels in an area to be changed.

Online APIs

  • As the main use case of Face-Off is to obfuscate faces on online platforms while preserving usability, experiments need to be conducted against online face recognition services and APIs.
  • For the scope of this course project, the results were constrained to 2 face recognition APIs (AWS, Azure).
  • To understand whether image scaling attacks will still perform on these online APIs, information about their model parameters (input size, face detector, scaling algorithm) needs to be extracted.
  • A dataset of image scaling attack faces was created with combinations of these parameters and was uploaded to both APIs.
